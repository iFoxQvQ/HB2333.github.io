前言

临近毕业，在制作自己简历的时候，觉得写的非常假大空，很难让人信服。在看了很多 V2EX 关于简历的帖子，觉得证明自己能力，应该要用更有效的、能查阅到的方式来证明。光说自己熟悉 xxx ，了解 xxx ，感觉自己都不相信自己。而阅读某个开源项目的源码，或参与开源项目社区、提交开源项目 PR，就足以证明在编码这一块的水平是足够的。而这一切的，就从如何进行 PR 开始。

PR流程

如果想在 Github 上提交一个 PR ，可以分为以下几个步骤：

  1. 先 Fork 别人的项目到自己的仓库里
  2. clone 到本地进行修改，修改的时候不会影响到别人的仓库，可以放心修改
  3. 修改完后，push 回自己的仓库
  4. 如果一切就绪，那么就可以提交 Pull Request 请求
  5. 当作者认同你的修改，那么他就会将你的代码合并到 master 上，而你也会成为这个项目的 contributor。

可能会遇到的问题

reviewers 提出修改意见

提交 PR 很多时候会因为各种各样的问题不会被马上合入，reviewers 会提出修改意见。这时你可以返回去继续根据意见修改代码，然后通过执行和第一次 Commit 一样步骤的命令，GitHub 会自动把新增的 commits 全部追加到一个未合入 PR 里去。
